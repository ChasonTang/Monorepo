# RFC-019: Event-Driven TCP Accept Loop

## 1. Summary

Add `odin/accept_loop.{c,h}`, an event-loop-driven multi-connection TCP accept listener: given a live `odin_event_loop_t` and a caller-owned `listen()`-ready nonblocking fd, `odin_accept_loop_create` registers one level-triggered READ watch on the listening fd through the RFC-010 `odin_event_io_*` API; on each readiness the loop drains pending connections by calling `accept4(SOCK_NONBLOCK)` (Linux) or `accept(2)` followed by `fcntl(F_SETFL, O_NONBLOCK)` (macOS) up to `ODIN_ACCEPT_LOOP_BATCH_MAX = 64` per readiness, invokes `on_accept(al, conn_fd, ud)` once per accepted nonblocking connection with the caller taking ownership of `conn_fd` at that moment; classifies `EAGAIN`/`EWOULDBLOCK` as the empty-queue drain exit, `EINTR`/`ECONNABORTED` as silent per-iteration drops, `EMFILE`/`ENFILE`/`ENOBUFS`/`ENOMEM` as a soft degradation that stops the READ watch and arms a 0-delay one-shot timer to re-arm the watch on the next loop pass (so a level-triggered EMFILE storm cannot hot-spin the CPU and the still-queued connections survive the pause), and every other accept errno as a one-shot `on_error(al, err, ud)` followed by a quiet terminal state in which the watch and timer stay stopped (no auto-destroy); `odin_accept_loop_destroy` stops the watch and any pending timer, never closes the caller-owned listening fd, and is safe to call from inside `on_accept` or `on_error`. The module depends only on `odin/event_loop.h`.

## 2. Goals

- **G1.** Given a level-triggered READ watch on the caller-supplied listening fd, deliver every pending TCP connection accepted from that fd to the caller through `on_accept(al, conn_fd, ud)` exactly once per `accept(2)` success, with `conn_fd` already set to nonblocking (so callers can attach the descriptor to the event loop without an extra `fcntl`) and caller ownership beginning at the callback's entry. Up to `ODIN_ACCEPT_LOOP_BATCH_MAX = 64` accepted connections are delivered per readiness; if more remain in the kernel queue, the level-triggered watch fires again on the next loop pass so no pending connection is permanently dropped by the cap. Per-iteration benign errnos `EAGAIN` / `EWOULDBLOCK` (empty queue) end the drain quietly, `EINTR` retries the next iteration's `accept(2)`, and `ECONNABORTED` skips the per-connection failure and continues the drain ŌĆö none of these surface as `on_error`.

- **G2.** Soft-degrade gracefully on `EMFILE` / `ENFILE` / `ENOBUFS` / `ENOMEM` from `accept(2)`: stop the READ watch on the listening fd and start a 0-delay one-shot `odin_event_timer_t`; the timer's callback re-arms the READ watch via a fresh `odin_event_io_start` so the very next loop pass picks up still-queued connections, preventing the level-triggered EMFILE storm that would otherwise hot-spin the loop on the listening fd while never closing the listener. The listening fd is never closed during soft degradation, so connections already in the kernel accept queue at the moment of the EMFILE failure are still acceptable once the file-descriptor table has room and the watch re-arms.

- **G3.** Aggregate any other `accept(2)` errno (e.g., `EBADF`, `ENOTSOCK`, `EOPNOTSUPP`, `EINVAL`, `EPERM`, `EPROTO`) and any failure to install or re-install the soft-degradation timer or the READ watch into a single `on_error(al, err, ud)` callback that fires exactly once with the failing errno, after which the accept loop is terminal: the READ watch and any pending timer are stopped before `on_error` is invoked, no further `on_accept` or `on_error` callback fires, and the module does not destroy itself ŌĆö the caller observes the failure, owns the still-open listening fd, and either closes/replaces the fd and creates a new `odin_accept_loop_t`, or invokes `odin_accept_loop_destroy(al)` to reclaim the handle. The constructor's local-setup failures (`ENOMEM` allocating the handle, or any errno from `odin_event_io_start`) instead return `-1` with `errno` set, leave `*out` untouched, and register no watch ŌĆö they never flow through `on_error`.

- **G4.** Give the caller lifetime control through `odin_accept_loop_destroy`: it stops the READ watch and any pending soft-degradation timer, frees the `odin_accept_loop_t` object, and never closes the caller-owned listening fd. It is safe to call from inside `on_accept` or `on_error` (the drain loop captures the callback target into locals before each invocation and checks a deferred-destroy flag on return, so the physical free runs after the in-flight callback unwinds, without `on_accept` reading the handle after destroy). `odin_accept_loop_destroy(NULL)` is a no-op.

## 3. Design

### 3.1 Overview

`odin/accept_loop` is a new leaf module in the Odin library. It depends only on `odin/event_loop.h`: it uses the RFC-010 `odin_event_io_*` API for the level-triggered READ watch on the listening fd and the `odin_event_timer_*` API for the 0-delay one-shot soft-degradation timer. It does not depend on `odin/transport`, `odin/dial`, `odin/relay`, or `odin/udp`, and it shares no code with `odin/client_listen` (whose `odin_client_listen_open` deliberately hard-binds to `127.0.0.1` for the Client side per the doc-comment at `odin/client_listen.h:8`; the Server side must listen on arbitrary addresses, so this module accepts an already-bound listening fd of the caller's choosing rather than building one).

The caller holds a live `odin_event_loop_t *` and a nonblocking `AF_INET` / `AF_INET6` / `AF_UNIX` listening fd it built itself (any `socket(2)` + `bind(2)` + `listen(2)` + `O_NONBLOCK` sequence, with whatever address-family and bind address the caller's product needs). It calls `odin_accept_loop_create(loop, listen_fd, on_accept, on_error, user_data, &al)`. The module registers one level-triggered READ watch on `listen_fd` through `odin_event_io_start` and returns. On each READ readiness, the module's internal `on_readable` runs an accept-drain loop bounded at `ODIN_ACCEPT_LOOP_BATCH_MAX = 64` iterations: each iteration issues one `accept4(listen_fd, NULL, NULL, SOCK_NONBLOCK)` on Linux or one `accept(listen_fd, NULL, NULL)` plus `fcntl(F_SETFL, O_NONBLOCK)` on macOS (the platform's atomic-nonblocking-accept primitive when available, the two-syscall fallback otherwise); on success it invokes `on_accept(al, conn_fd, user_data)` with the caller taking ownership of `conn_fd` from that moment; on `EAGAIN`/`EWOULDBLOCK` it exits the drain (level-triggered READ re-fires on the next pass if more connections arrive); on `EINTR` it retries the next iteration's accept call; on `ECONNABORTED` it skips this connection and continues the drain; on `EMFILE` / `ENFILE` / `ENOBUFS` / `ENOMEM` it enters soft degradation (stops the READ watch, starts a 0-delay one-shot timer through `odin_event_timer_start`, returns); on any other errno it transitions to a terminal state (stops watch and timer if any, fires `on_error(al, err, user_data)` exactly once, does not free the handle). When the soft-degradation timer fires, the module re-arms the READ watch via a fresh `odin_event_io_start` ŌĆö if that re-arm fails (e.g., kqueue `ENOSPC`), it falls through to the same terminal `on_error` path. The module never closes `listen_fd`, never closes a `conn_fd` after handing it to `on_accept`, and never destroys itself; lifetime ends only at the caller's `odin_accept_loop_destroy(al)`.

```
       caller                       odin_accept_loop_t                        event loop
   nonblocking listen_fd ---odin_accept_loop_create---> READ watch (odin_event_io_start)
                                                  on READ readiness:
                                                     drain loop, up to 64 iterations:
                                                        accept4 / accept+fcntl
                                                          OK         -> on_accept(al, conn_fd, ud)   // caller owns conn_fd
                                                          EAGAIN     -> exit drain   (level-triggered re-fires)
                                                          EINTR      -> retry next iteration
                                                          ECONNABORTED -> skip; continue drain
                                                          EMFILE/ENFILE/ENOBUFS/ENOMEM ->
                                                                       odin_event_io_stop(io); io = NULL
                                                                       odin_event_timer_start(loop, 0, 0)        \
                                                                                                                  > soft degradation
                                                                       (next loop pass) on_resume_timer:          /
                                                                          odin_event_timer_stop(timer); timer = NULL
                                                                          odin_event_io_start(loop, listen_fd, READ, ...)
                                                                          (on failure -> on_error)
                                                          other errno -> stop watch+timer; on_error(al, err, ud) once; terminal
                                                                          (no free; no further callbacks)

   odin_accept_loop_destroy(al):   stop watch+timer; never closes listen_fd; safe from inside on_accept / on_error
                                   (deferred free while a callback is on the stack)
```

### 3.2 Detailed Design

#### 3.2.1 Public API, Ownership, and Lifecycle

```c
/* odin/accept_loop.h */
#include "odin/event_loop.h"

#ifdef __cplusplus
extern "C" {
#endif

#define ODIN_ACCEPT_LOOP_BATCH_MAX 64u

typedef struct odin_accept_loop_t odin_accept_loop_t;

typedef void (*odin_accept_loop_accept_cb)(odin_accept_loop_t *al, int conn_fd,
                                           void *user_data);

typedef void (*odin_accept_loop_error_cb)(odin_accept_loop_t *al, int err,
                                          void *user_data);

int odin_accept_loop_create(odin_event_loop_t *loop, int listen_fd,
                            odin_accept_loop_accept_cb on_accept,
                            odin_accept_loop_error_cb on_error, void *user_data,
                            odin_accept_loop_t **out);

void odin_accept_loop_destroy(odin_accept_loop_t *al);

#ifdef __cplusplus
}
#endif
```

**Unstated contract.**

- **Ownership and preconditions.** `loop`, `on_accept`, `on_error`, and `out` are non-null preconditions. `listen_fd` is a caller-owned, already-`bind(2)` + `listen(2)` socket that must be nonblocking (`O_NONBLOCK` set) before the call ŌĆö the module does not mutate the listening fd's flags, in line with the dial/relay "caller owns fd" convention and with the RFC-010 contract that `odin_event_io_start` watches "caller-owned nonblocking fd"s (`odin/event_loop.h:51-53`). Address family and bind address are entirely the caller's choice (this RFC works for `AF_INET`, `AF_INET6`, and `AF_UNIX` listening sockets); the module does not parse, validate, or rewrite the bound address, which is what lets a server-side caller listen on an interface other than the hard-wired `127.0.0.1` that `odin_client_listen_open`'s doc-comment at `odin/client_listen.h:8` pins. `user_data` is opaque to the module and is passed verbatim to every `on_accept` and `on_error` invocation. `odin_accept_loop_create` is an owner-thread API under the RFC-010 contract ŌĆö callable before `run`, from a loop callback, or from a posted task.
- **Success.** On success `odin_accept_loop_create` allocates the `odin_accept_loop_t`, registers exactly one level-triggered `ODIN_EVENT_READ` watch on `listen_fd` via `odin_event_io_start`, writes `*out`, and returns `0`. It never invokes `on_accept` or `on_error` itself ŌĆö both callbacks fire only from a later loop turn driven by an actual `accept(2)` outcome.
- **Failure and rollback.** A local setup failure returns `-1` with `errno` set, writes nothing to `*out`, and registers no watch: `ENOMEM` when the `odin_accept_loop_t` cannot be allocated, or the errno from `odin_event_io_start` (e.g. `EEXIST` if the loop already has an active watch on `listen_fd`, `EINVAL` for an invalid input mask ŌĆö which this module never triggers because it passes `ODIN_EVENT_READ` ŌĆö or backend allocation errnos like `ENOMEM`). After any setup failure the handle is freed and no callback ever fires; the caller's `listen_fd` is untouched. `accept(2)` failures from a later readiness are never reported through this return path ŌĆö they always flow through `on_error` (┬¦3.2.2 / ┬¦3.2.3).
- **`on_accept` completion contract.** Fires zero or more times on the owner thread, once per successful `accept(2)`. `conn_fd` is a freshly accepted stream socket that the module has already made nonblocking (via `accept4(SOCK_NONBLOCK)` where available, or `accept(2)` + `fcntl(F_SETFL, O_NONBLOCK)` otherwise ŌĆö ┬¦3.2.2 Mechanism), with no other socket options set. Caller ownership begins at the moment of the call: the module will never close, read from, write to, or otherwise touch `conn_fd` again. If `on_accept` does not retain `conn_fd`, the descriptor leaks ŌĆö this is the caller's responsibility. `on_accept` may call `odin_accept_loop_destroy(al)` (the drain loop captures the callback target and user data into locals and checks a deferred-destroy flag on return ŌĆö see *Destroy*), but it must not perform a `setrlimit` or other action that itself raises new `EMFILE`s under the same drain iteration; benign reentrancy beyond `destroy` is permitted because the module reads no `al` state between the callback's return and the post-callback flag check.
- **`on_error` completion contract.** Fires at most once on the owner thread. `err` is the failing errno of the syscall that caused the terminal transition (a non-soft-degradation `accept(2)` errno; the errno from a failed `odin_event_timer_start` during soft-degradation entry; or the errno from a failed `odin_event_io_start` re-arm at the end of soft degradation). Before `on_error` is invoked, the READ watch is stopped (`io == NULL`) and any pending soft-degradation timer is stopped (`timer == NULL`) ŌĆö so the listener exposes no more readiness or timer triggers from this RFC. The module does not destroy itself, does not close `listen_fd`, and after `on_error` returns reads only the `destroy_requested` flag ŌĆö the deferred-destroy contract keeps `al` alive across that read (the callback target and user data are captured into locals before the call, mirroring the dial's `complete` discipline in `odin/dial.c:62-83`); destroy from inside `on_error` is therefore safe and is the typical caller response. After `on_error` returns and the in-callback destroy check (┬¦3.2.3) runs, the handle stays alive in a *terminal* state until the caller invokes `odin_accept_loop_destroy(al)`; no `on_accept` or `on_error` callback fires from that terminal state.
- **Destroy.** `odin_accept_loop_destroy(NULL)` is a no-op. `odin_accept_loop_destroy(al)` outside a callback stops any still-active READ watch (`odin_event_io_stop`) and any still-active soft-degradation timer (`odin_event_timer_stop`), frees the handle, and returns; the listening fd is **never** closed (the requirement: "accept_loop_destroy µ│©ķöĆ watch’╝īõĖŹÕģ│ listening fd"). Called from inside `on_accept` or `on_error`, it sets a `destroy_requested` flag and returns without freeing; the drain loop or the terminal-error path that captured the callback target into locals notices the flag after the callback returns and runs the physical free then, so the handle is never touched after the callback returns from within `on_accept` / `on_error`. The pointer is dead after the physical free. `destroy` never invokes `on_accept` or `on_error`.
- **Threading.** All entry points and internal callbacks (`on_readable`, `on_resume_timer`) run on the loop's owner thread; the module adds no locks.

**Mechanism.**

```
create(loop, listen_fd, on_accept, on_error, user_data, out):
  al = calloc(1, sizeof *al)
  if al == NULL: errno = ENOMEM; return -1
  al.loop = loop; al.listen_fd = listen_fd
  al.on_accept = on_accept; al.on_error = on_error; al.user_data = user_data
  al.io = NULL; al.timer = NULL
  al.in_callback = 0; al.destroy_requested = 0; al.terminal = 0
  if odin_event_io_start(loop, listen_fd, ODIN_EVENT_READ, on_readable, al, &al.io) != 0:
    saved = errno; free(al); errno = saved; return -1
  *out = al; return 0

destroy(al):
  if al == NULL: return
  if al.in_callback:
    al.destroy_requested = 1                  /* finish_destroy runs after the callback returns */
    return
  finish_destroy(al)

finish_destroy(al):                            /* never closes listen_fd */
  if al.io != NULL:    odin_event_io_stop(al.io);       al.io = NULL
  if al.timer != NULL: odin_event_timer_stop(al.timer); al.timer = NULL
  free(al)
```

Satisfies: G1 via the public `odin_accept_loop_accept_cb` surface that delivers each accepted `conn_fd` with caller ownership; G2 via the watch+timer state the ┬¦3.2.3 soft-degradation transitions manipulate through this contract; G3 via the public `odin_accept_loop_error_cb` surface plus the constructor's `-1`/`errno` local-setup-failure rollback that registers no watch and writes nothing to `*out`; G4 via the caller-owned `odin_accept_loop_t` handle, the never-close-`listen_fd` rule, and the `in_callback` / `destroy_requested` deferred-free dance that makes `destroy` from inside `on_accept` or `on_error` safe.

#### 3.2.2 Accept Drain Loop and Errno Classification

```c
/* odin_accept_loop_t internal state (Proposed). */
struct odin_accept_loop_t {
  odin_event_loop_t *loop;
  int listen_fd;                          /* caller-owned; never closed by this module */
  odin_accept_loop_accept_cb on_accept;
  odin_accept_loop_error_cb on_error;
  void *user_data;
  odin_event_io_t *io;                    /* READ watch; NULL while paused or terminal       */
  odin_event_timer_t *timer;              /* 0-delay one-shot soft-degradation timer or NULL  */
  int in_callback;                        /* 1 while inside on_accept or on_error             */
  int destroy_requested;                  /* 1 if destroy was called during a callback        */
  int terminal;                           /* 1 once on_error has fired                        */
#if defined(ODIN_ACCEPT_LOOP_TESTING)
  int fail_next_accept_err;               /* one-shot synthetic errno for accept_one's accept step; 0 = unarmed */
#if !defined(__linux__)
  int fail_next_fcntl_which;              /* F_GETFL or F_SETFL when a fcntl fault is armed; 0 = unarmed         */
  int fail_next_fcntl_err;                /* one-shot synthetic errno for the matching fcntl step                */
#endif
#endif
};
```

**Unstated contract.**

- **Atomic nonblocking accept where the platform offers it.** On Linux the module issues `accept4(listen_fd, NULL, NULL, SOCK_NONBLOCK)` so the accepted fd is nonblocking before any other syscall observes it. On macOS ŌĆö where `accept4(2)` is unavailable ŌĆö it issues `accept(listen_fd, NULL, NULL)` followed by `fcntl(fd, F_SETFL, fcntl(fd, F_GETFL) | O_NONBLOCK)`; if the `fcntl` step fails (rare; would-be `ENOMEM`-class) the accepted fd is closed (caller never sees a half-configured fd) and the `fcntl`-returned errno is fed back through the same classification arms below as if it had come from the `accept(2)` itself, so `EMFILE` / `ENFILE` / `ENOBUFS` / `ENOMEM` from `fcntl` enter soft degradation and everything else routes to `on_error`. Either way, every `conn_fd` handed to `on_accept` is nonblocking. The module does not call `getpeername(2)`, request the peer address from `accept(2)` (the `addr` / `addrlen` arguments are `NULL`), or set any other socket option on the accepted fd ŌĆö callers that need peer addresses or `TCP_NODELAY` etc. set them in `on_accept` from the fd they now own.
- **Drain bound and re-arm.** Each `on_readable` performs at most `ODIN_ACCEPT_LOOP_BATCH_MAX = 64` `accept(2)` calls before returning to the event loop. The READ watch is level-triggered (RFC-010 `odin_event_io_start` registers a "level-triggered watch", `odin/event_loop.h:51-53`), so if the kernel's accept queue is non-empty when the drain returns the next loop pass observes another READ readiness and resumes draining ŌĆö no pending connection is lost to the cap. The drain returns early (before exhausting the 64 budget) when (a) `accept(2)` returns `EAGAIN` / `EWOULDBLOCK` (queue is empty ŌĆö wait for the next readiness), (b) a soft-degradation errno fires (queue retained but module pauses), (c) a terminal errno fires (`on_error` invoked, module quiesces), or (d) `on_accept` requested destroy via `odin_accept_loop_destroy(al)`. `EINTR` and `ECONNABORTED` each count as one iteration toward the 64 budget ŌĆö this is deliberate, so a peer that floods the listener with RSTs (yielding `ECONNABORTED` per accept) cannot starve the loop by spending all 64 iterations on failed accepts that never count.
- **Benign errno classification.** `EAGAIN` and `EWOULDBLOCK` are the empty-queue signal and end the drain quietly ŌĆö no `on_accept`, no `on_error`. `EINTR` means the `accept(2)` syscall was interrupted by a signal before doing any work; the drain advances one iteration and the next iteration re-issues `accept(2)` on the same fd. `ECONNABORTED` means the peer issued an RST between landing in the accept queue and the kernel handing us the fd, so there is no `conn_fd` to deliver; the drain skips this iteration (no `on_accept`, no `on_error`) and continues to the next, matching the standard Unix idiom that lets a per-connection RST not kill the listener. None of these three errnos surface to `on_error`. **Deviation from the literal requirement.** The original requirement classifies errors as `EAGAIN`/`EINTR` ŌåÆ silent retry, `EMFILE`/`ENFILE`/`ENOBUFS`/`ENOMEM` ŌåÆ soft degradation, "ÕģČõĮÖ ŌåÆ `on_error`" ŌĆö a literal reading would route `ECONNABORTED` to `on_error`. This RFC intentionally places `ECONNABORTED` in the silent-drop arm instead, because terminating the listener on every peer RST would let any client trivially DoS the server by issuing `connect`+`RST` pairs; the Unix idiom above is the load-bearing reason, and ┬¦5 T5 asserts the resulting behavior.
- **Soft-degradation triggers.** `EMFILE` (per-process fd-table exhaustion), `ENFILE` (system-wide fd-table exhaustion), `ENOBUFS` (socket-buffer memory exhaustion), and `ENOMEM` (kernel memory exhaustion) all map to the ┬¦3.2.3 *soft degradation* entry: the READ watch is stopped immediately so the level-triggered backend cannot keep firing on the still-non-empty accept queue (which would cause the canonical EMFILE hot-spin), and a 0-delay one-shot timer is started to re-arm the watch on the next loop pass. The listening fd is never closed during this transition, so the queued connections survive until the watch re-arms.
- **Terminal errnos.** Every other `accept(2)` errno ŌĆö including but not limited to `EBADF` (`listen_fd` was closed under us), `ENOTSOCK` (`listen_fd` is not a socket), `EOPNOTSUPP` (socket not listening or wrong type), `EINVAL` (invalid argument), `EPERM` (firewall), `EPROTO` (protocol error), and any platform-specific outlier ŌĆö routes to the ┬¦3.2.3 terminal transition: the watch is stopped (no further `accept` attempts), any pending soft-degradation timer is stopped (none should be active in this branch, but `enter_terminal_error` is symmetric for the case where re-arm itself failed), and `on_error(al, err, user_data)` fires exactly once. The module does not auto-destroy ŌĆö the caller decides whether to retry construction with a replacement fd or to call `odin_accept_loop_destroy(al)`.

**Mechanism.**

```
ODIN_ACCEPT_LOOP_BATCH_MAX = 64

accept_one(al, &conn_fd, &err):                                       /* returns 0 / OK, 1 / FAIL */
#if defined(ODIN_ACCEPT_LOOP_TESTING)
  if al.fail_next_accept_err != 0:                                    /* ┬¦3.2.4 _test_fail_next_accept */
    err = al.fail_next_accept_err
    al.fail_next_accept_err = 0
    return FAIL                                                       /* skips the real accept4 / accept syscall */
#endif
#if defined(__linux__)
  r = accept4(al.listen_fd, NULL, NULL, SOCK_NONBLOCK)
  if r >= 0: conn_fd = r; return OK
  err = errno; return FAIL
#else  /* macOS and other platforms without accept4 */
  r = accept(al.listen_fd, NULL, NULL)
  if r < 0: err = errno; return FAIL
#if defined(ODIN_ACCEPT_LOOP_TESTING)
  if al.fail_next_fcntl_which == F_GETFL:                             /* ┬¦3.2.4 _test_fail_next_fcntl, F_GETFL variant */
    saved = al.fail_next_fcntl_err
    al.fail_next_fcntl_which = 0; al.fail_next_fcntl_err = 0
    close(r); err = saved; return FAIL                                /* skips both real fcntl syscalls */
#endif
  flags = fcntl(r, F_GETFL, 0)
  if flags == -1:
    saved = errno; close(r); err = saved; return FAIL
#if defined(ODIN_ACCEPT_LOOP_TESTING)
  if al.fail_next_fcntl_which == F_SETFL:                             /* ┬¦3.2.4 _test_fail_next_fcntl, F_SETFL variant */
    saved = al.fail_next_fcntl_err
    al.fail_next_fcntl_which = 0; al.fail_next_fcntl_err = 0
    close(r); err = saved; return FAIL                                /* the F_GETFL step above ran for real */
#endif
  if fcntl(r, F_SETFL, flags | O_NONBLOCK) == -1:
    saved = errno; close(r); err = saved; return FAIL
  conn_fd = r; return OK
#endif

on_readable(loop, io, fd, events, user_data):    /* user_data = al */
  al = user_data
  if al.terminal: return                          /* defensive; watch is already stopped here */
  for i in [0, ODIN_ACCEPT_LOOP_BATCH_MAX):
    if accept_one(al, &conn_fd, &err) == OK:
      cb = al.on_accept; ud = al.user_data        /* capture before the call (mirrors enter_terminal_error) */
      al.in_callback = 1
      cb(al, conn_fd, ud)
      if al.destroy_requested:                    /* destroy was called from on_accept */
        finish_destroy(al); return
      al.in_callback = 0
      continue
    /* accept failed; classify err */
    if err == EAGAIN or err == EWOULDBLOCK: return
    if err == EINTR or err == ECONNABORTED: continue
    if err == EMFILE or err == ENFILE or err == ENOBUFS or err == ENOMEM:
      enter_soft_degradation(al); return
    enter_terminal_error(al, err); return
```

Satisfies: G1 via the per-iteration `accept_one -> on_accept` delivery and the `EAGAIN` / `EINTR` / `ECONNABORTED` benign-errno arms that never surface to `on_error`; G2 via the `EMFILE` / `ENFILE` / `ENOBUFS` / `ENOMEM` arm that routes into `enter_soft_degradation` and the watch+queue invariants the bullet above pins; G3 via the catch-all `enter_terminal_error(al, err)` arm for every other errno and the never-auto-destroy contract that lets the caller dispose of the handle; G4 via the `in_callback` / `destroy_requested` check after each `on_accept` invocation that runs `finish_destroy` exactly when the callback asked for destroy and returns immediately afterwards, so the drain reads no `al` state after the freed handle.

#### 3.2.3 Soft Degradation, Terminal Error, and Deferred Destroy

State machine:

```text
State        Trigger                                       Next State   Observable action
-----------  --------------------------------------------  -----------  -----------------------------------------------
ACTIVE       create succeeded                              ACTIVE       READ watch armed
ACTIVE       accept OK                                     ACTIVE       on_accept(al, conn_fd, ud)
ACTIVE       EAGAIN / EWOULDBLOCK                          ACTIVE       drain exits; level-triggered re-fires later
ACTIVE       EINTR / ECONNABORTED                          ACTIVE       skip iteration; drain continues
ACTIVE       EMFILE / ENFILE / ENOBUFS / ENOMEM            PAUSED       stop watch; start 0-delay one-shot timer
ACTIVE       timer-start fails during PAUSED entry         TERMINAL     watch already stopped; on_error(al, err, ud)
ACTIVE       other accept errno                            TERMINAL     stop watch; on_error(al, err, ud)
ACTIVE       destroy outside callback                      DEAD         stop watch; free handle (listen_fd untouched)
ACTIVE       destroy inside on_accept                      DEAD         deferred free after callback returns

PAUSED       timer fires; re-arm READ watch fails          TERMINAL     timer stopped; on_error(al, err, ud)
PAUSED       destroy outside callback                      DEAD         stop timer; free handle (listen_fd untouched)
TERMINAL     destroy outside callback                      DEAD         free handle (watch + timer already stopped)
TERMINAL     destroy inside on_error                       DEAD         deferred free after callback returns
```

**Unstated contract.**

- **Soft degradation entry is failure-closed and queue-preserving.** `enter_soft_degradation` stops the READ watch *first* (so the level-triggered backend cannot re-fire on the still-non-empty accept queue while we are mid-recovery and hot-spin the loop), then starts the 0-delay one-shot timer. The 0-delay timer fires on the next timer pass per RFC-010 (`delay_us==0` is "due on the next timer pass", `odin/event_loop.h:72-73`), so the recovery latency is exactly one event-loop iteration ŌĆö long enough to give the rest of the loop a chance to run a posted task or another fd's callback that releases an fd (the typical EMFILE remedy) before we attempt another `accept(2)`. If `odin_event_timer_start` itself fails (its only documented runtime failure is `ENOMEM`), the module cannot recover automatically and the request flows into `enter_terminal_error(al, err)` with that timer-start errno; this is the same surface a non-soft-degradation `accept` errno reaches, so callers see one uniform error path.
- **Soft degradation exit re-arms with `odin_event_io_start` (not `odin_event_io_update`).** The watch was *stopped* on entry (not merely toggled to an empty mask), so the previous `odin_event_io_t *` was released; recovery installs a *fresh* watch through `odin_event_io_start(loop, listen_fd, ODIN_EVENT_READ, on_readable, al, &al.io)`. The fresh watch fires on the very next loop pass if the kernel still has queued connections (level-triggered backend), which is precisely how the requirement's "µ£¤ķŚ┤ÕĘ▓ pending ńÜäĶ┐×µÄźõĖŹõĖó" property is satisfied. If `odin_event_io_start` fails on re-arm (e.g. macOS kqueue returns `ENOSPC` from a backend kevent change failure), the module flows into `enter_terminal_error(al, err)` with that re-arm errno ŌĆö same caller-visible surface as a non-soft-degradation accept errno or a timer-start failure.
- **Terminal entry stops both registrations before firing.** `enter_terminal_error(al, err)` stops both `al.io` and `al.timer` (each guarded by a NULL check, so it is safe whether the transition arrives from the active path, from a failed soft-degradation entry, or from a failed soft-degradation exit) *before* invoking `on_error`. After `on_error` is invoked the module is *terminal*: `al.terminal == 1`, no further readiness or timer events are possible from this RFC's surface, and a stray invocation of `on_readable` from a backend race (none expected, but cheap to defend) early-returns on the `terminal` check. The callback is invoked from `enter_terminal_error` only after the callback target and user data are captured into locals (mirroring the dial's `complete` discipline in `odin/dial.c:72-82`), so destroy-from-inside-`on_error` is safe ŌĆö after the callback returns the module reads only `al.destroy_requested`, and the deferred-destroy contract keeps `al` alive across that read.
- **`in_callback` / `destroy_requested` deferred destroy.** Both `on_readable`'s per-iteration `on_accept` call and `enter_terminal_error`'s `on_error` call sandwich the callback between `al.in_callback = 1` (set just before the call) and a post-callback `if al.destroy_requested: finish_destroy(al); return` (run as the first statement after control returns). An `odin_accept_loop_destroy(al)` invoked from within either callback sees `al.in_callback == 1`, sets `al.destroy_requested = 1`, and returns without freeing; the caller-supplied callback returns; the outer code observes the flag and runs `finish_destroy(al)`, which stops whatever watch or timer was still active and frees the handle. This pattern is identical in shape to the dial's "final action" rule (`odin/dial.c:62-83`) and the connect-session's "captured cb/ud before invocation" rule (RFC-018 ┬¦3.2.4 `fire_on_done_once`).
- **No backpressure feedback.** Once `on_error` fires, the module makes no further attempt at recovery, even if the original errno (`EBADF`, etc.) was hypothetically transient. The caller is the only entity that knows what to do (e.g. replace `listen_fd` with a freshly built listener and create a new `odin_accept_loop_t`, vs. give up). This matches G3's no-auto-destroy clause.

**Mechanism.**

```
enter_soft_degradation(al):
  /* invariant on entry: al.io != NULL && al.timer == NULL && !al.terminal */
  odin_event_io_stop(al.io); al.io = NULL                    /* stop the level-triggered storm   */
  if odin_event_timer_start(al.loop, 0, 0, on_resume_timer, al, &al.timer) != 0:
    err = errno
    enter_terminal_error(al, err)                            /* timer install failed; surface it */
    return

on_resume_timer(loop, timer, user_data):                     /* user_data = al */
  al = user_data
  odin_event_timer_stop(al.timer); al.timer = NULL
  if odin_event_io_start(al.loop, al.listen_fd, ODIN_EVENT_READ, on_readable, al, &al.io) != 0:
    err = errno
    enter_terminal_error(al, err)                            /* re-arm failed; surface it */
    return
  /* watch is back; next readiness resumes the drain in on_readable. */

enter_terminal_error(al, err):
  if al.io != NULL:    odin_event_io_stop(al.io);       al.io = NULL
  if al.timer != NULL: odin_event_timer_stop(al.timer); al.timer = NULL
  al.terminal = 1
  al.in_callback = 1
  cb = al.on_error; ud = al.user_data                        /* capture before the call */
  cb(al, err, ud)
  if al.destroy_requested:
    finish_destroy(al); return
  al.in_callback = 0
```

Satisfies: G2 via `enter_soft_degradation`'s stop-watch-then-start-timer sequence (the watch-stop step breaks the level-triggered EMFILE hot-spin) and `on_resume_timer`'s stop-timer-then-fresh-`odin_event_io_start` re-arm that picks up still-queued connections on the next loop pass ŌĆö neither step closes `listen_fd`; G3 via `enter_terminal_error`'s stop-both-then-set-`terminal`-then-capture-`cb`/`ud`-then-fire-once discipline that funnels every non-soft-degradation accept errno, the timer-start failure from `enter_soft_degradation`, and the re-arm failure from `on_resume_timer` into the same single `on_error(al, err, ud)` and leaves the module quiesced without auto-destroy; G4 via the `in_callback` / `destroy_requested` deferred-free dance after `on_error` that makes `odin_accept_loop_destroy` from inside `on_error` safe, mirroring the `on_accept` deferral in ┬¦3.2.2.

## 4. Security

The module accepts TCP connections from arbitrary network peers (when the caller binds a public address; loopback-only callers face only local peers but the trust boundary is identical in shape). The peer's *bytes* never reach the module ŌĆö the module only invokes `accept(2)` and hands the resulting `conn_fd` upward; peer-level parsing is the caller's responsibility. The peer's *behavior*, however, can trigger system resource exhaustion at the `accept(2)` syscall through the rate and pattern of `connect(2)` attempts, which is a real attack surface this RFC introduces by exposing a long-lived `accept(2)` loop.

- **S1.**
  - **Threat:** A peer (or many peers) saturates the listener's connection rate to the point that `accept(2)` returns `EMFILE` (per-process fd-table full) or `ENFILE` / `ENOBUFS` / `ENOMEM` (system-wide exhaustion). Under a naive level-triggered READ watch + plain `accept(2)` loop, the kernel keeps reporting READ ready (because the accept queue is non-empty), the userspace loop keeps calling `accept(2)` and getting the same exhaustion errno, and the loop hot-spins 100% of one CPU until the file-descriptor situation changes. This is a denial-of-service multiplier: the listener wastes CPU it should be using to *drain* fds elsewhere (which is what would relieve `EMFILE`), and other I/O on the same loop is starved. Calling `on_error` and quitting also abandons the still-pending real connections sitting in the kernel's accept queue.
  - **Mitigation:** ┬¦3.2.2's classification routes `EMFILE` / `ENFILE` / `ENOBUFS` / `ENOMEM` into ┬¦3.2.3's `enter_soft_degradation`, which stops the READ watch *before* starting a 0-delay one-shot timer. Stopping the watch is the load-bearing step: with no watch on `listen_fd`, the level-triggered backend cannot re-fire on the still-non-empty accept queue, breaking the hot-spin. The 0-delay timer hands control back to the event loop for one pass ŌĆö enough for any other ready callback to release fds ŌĆö and then re-arms a fresh READ watch via `odin_event_io_start`; if the kernel queue is still non-empty, the re-armed watch fires on the very next pass and the drain resumes. The listening fd is never closed, so queued connections survive the pause.
  - **Enforcement:** Row T6 (real `setrlimit(RLIMIT_NOFILE)` trigger) connects N+1 peers (where N exhausts the budget), runs the loop, and asserts that on the EMFILE-hitting readiness the module enters paused state (`odin_accept_loop_test_is_paused(al) == 1`, no further `on_readable` invocations between pause and timer fire), then ŌĆö once the test releases an fd to make room ŌĆö that the timer fires, re-arms the watch, and `on_accept` delivers the previously-queued surplus connection without the test having had to close and re-open the listening fd. Row T7 exercises each of `ENFILE` / `ENOBUFS` / `ENOMEM` via the ┬¦3.2.4 test hook and asserts the same paused-then-recover transition, so a regression that handled `EMFILE` correctly but mishandled (or did not classify) any of the other three would fail.

## 5. Testing Strategy

Rows `T1`ŌĆō`T18` are unit tests in `odin/accept_loop_unittests.cpp` and execute under the same `fork` + `waitpid` 2 s deadline fixture RFC-012 ┬¦6 established (`DialRunDeadline` in `odin/dial_unittests.cpp:42-73`, replicated as `AcceptLoopRunDeadline`): the child runs the row body and all `ASSERT_*` / `EXPECT_*`, then `_exit(::testing::Test::HasFailure() ? 1 : 0)`, and the parent fails the row unless the child exits `0` within the deadline. Rows that enter `odin_event_loop_run` ŌĆö every row except T16 ŌĆö additionally arm a one-shot 200 ms watchdog `odin_event_timer` that sets `state.timed_out = true` and calls `odin_event_loop_stop`, so a row whose `on_accept` / `on_error` never fires cannot hang the child; T16 is the synchronous constructor-rollback row that never calls `odin_event_loop_run` and therefore relies on the parent's 2 s deadline alone to bound a hypothetical constructor hang.
